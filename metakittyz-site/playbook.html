Netlify Functions — API Architecture

POST /api/anthropic
Primary AI endpoint. Calls claude-opus-4-8. Includes 3-retry logic with 2s → 4s → 8s exponential backoff. Returns JSON with agent response + token usage.
POST /api/openai
Fallback AI endpoint. Activates automatically if Anthropic endpoint fails after 3 retries. Uses gpt-4o. Same request/response shape as primary.
POST /api/guardian
QA scoring endpoint. Accepts any agent output and returns a Guardian score 0–100. Score ≥70 = APPROVED. 40–69 = WARN. <40 = BLOCK.

netlify.toml Configuration

[build] functions = "netlify/functions" publish = "." [[headers]] for = "/api/*" [headers.values] Access-Control-Allow-Origin = "https://[BRAND_DOMAIN]" Access-Control-Allow-Methods = "POST, OPTIONS" Access-Control-Allow-Headers = "Content-Type" [context.production.environment] NODE_VERSION = "18"

API Cascade Logic (Client-Side)

async function callAI(prompt) { // Step 1: Try Anthropic (3 retries) for (let i = 0; i < 3; i++) { try { const res = await fetch('/api/anthropic', { method: 'POST', headers: {'Content-Type':'application/json'}, body: JSON.stringify({ prompt }) }); if (res.ok) return await res.json(); } catch(e) { await sleep([2000, 4000, 8000][i]); // exponential backoff } } // Step 2: Fallback to OpenAI const res = await fetch('/api/openai', { method: 'POST', headers: {'Content-Type':'application/json'}, body: JSON.stringify({ prompt }) }); return await res.json(); }

§05
The 6-Agent AI System

Six specialized AI agents. Each has one job, strict constraints, and cannot skip the pipeline order. MATRIX starts every request. GUARDIAN ends every request. No exceptions.

🔒 Pipeline Rule: MATRIX → NOVA → SPARK → PULSE → ARIA → GUARDIAN. Always in this order. No agent can be skipped. No agent can pass output directly to a non-adjacent agent in the chain.
🎯
MATRIX
POSITION 1 OF 6 · STRATEGY DIRECTOR
claude-opus-4-8
  • FIRST agent in every pipeline run — no exceptions
  • Analyzes goal, audience, platform algorithm signals
  • Outputs a strategy brief: topic, angle, hook hypothesis
  • Cannot produce final content — strategy only
  • If MATRIX output is rejected by GUARDIAN: full restart
  • Max output: 400 words. No fluff, no padding.
📊
NOVA
POSITION 2 OF 6 · RESEARCH ANALYST
claude-opus-4-8
  • Receives MATRIX strategy brief as input only
  • Researches trends, data points, competitor analysis
  • Outputs a research dossier: 5–8 verified facts/stats
  • Cannot make creative decisions — data only
  • All claims must include source category (e.g., "industry study")
  • Max output: 600 words. Bullet-point format preferred.
⚡
SPARK
POSITION 3 OF 6 · VIRAL CONTENT STRATEGIST
claude-opus-4-8
  • Receives NOVA research dossier as input
  • Generates hook, title, script outline, thumbnail concept
  • Scores own output on 40-point viral scale (minimum pass: 28/40)
  • If score < 28: regenerates up to 2 more times, then flags
  • Viral scale: Hook(10) + Retention(10) + Shareability(10) + CTA(10)
  • Cannot produce final media — creative brief only
🎬
PULSE
POSITION 4 OF 6 · CONTENT PRODUCER
claude-opus-4-8
  • Receives SPARK creative brief as input
  • Writes full script: intro hook, body, outro CTA
  • Produces short-form clip titles and descriptions
  • Generates stream segment timestamps
  • Cannot publish — production only, hands off to ARIA
  • Max script length: 1500 words (≈ 8–10 min video)
📡
ARIA
POSITION 5 OF 6 · DISTRIBUTION MANAGER
claude-opus-4-8
  • Receives PULSE script + metadata as input
  • Generates platform-specific posts: Twitter/X, TikTok, Instagram, Discord
  • Creates stream schedule and cross-promotion calendar
  • Adapts tone per platform — cannot use identical copy across platforms
  • Cannot publish directly — outputs copy for human review
  • Flags any content that may violate platform TOS before GUARDIAN
🛡
GUARDIAN
POSITION 6 OF 6 · QA GATEKEEPER
gpt-4o (independent)
  • LAST agent in every pipeline run — cannot be skipped
  • Uses gpt-4o (intentionally different model — prevents bias)
  • Scores all ARIA output 0–100 across 5 dimensions
  • Score ≥ 70: APPROVED ✅ · 40–69: WARN ⚠️ · <40: BLOCK ❌
  • BLOCK = content cannot be published, returns to MATRIX
  • Logs every score + reason in the session report card
⚡ Critical Rule: GUARDIAN uses a different AI model (gpt-4o) than the rest of the pipeline (claude-opus-4-8). This is intentional — an independent model reviewing output catches blind spots that the primary model might share. Never change GUARDIAN to the same model as the other agents.

Safety & Security Setup

Data security is non-negotiable. Every API call, every piece of user data, every environment variable — here's how to keep it locked.

Security Checklist

S1
API Keys — Never in Code
All API keys live in Netlify environment variables only. Never in HTML, JS files, or committed to GitHub. If you accidentally commit a key, rotate it immediately at console.anthropic.com.
✓ Run: git grep "sk-ant" — output should be empty
CRITICAL
S2
CORS — Lock API to Your Domain
Your Netlify functions should only accept requests from your domain. The Access-Control-Allow-Origin header in netlify.toml (§03) handles this. Anyone else hitting your API endpoint will be rejected.
✓ Test: from a different domain, your /api/anthropic returns a CORS error
HIGH
S3
Rate Limiting
Add a basic rate limit to your Netlify functions: max 10 requests per IP per minute. This prevents abuse and runaway API costs. Use a simple in-memory Map for tracking (works fine for low traffic).
⚠ Without rate limiting, a single malicious actor could drain your entire AI API budget in minutes
HIGH
S4
Billing Alerts
Set spending limits in Anthropic dashboard (console.anthropic.com → Billing) and OpenAI dashboard. Start at $50/month cap. Set email alert at 80% of cap. Increase only when revenue supports it.
✓ Email alert fires a test notification when you manually trigger it
MEDIUM
S5
Email Collection — Compliance
If you collect emails (like the Learn More form on about.html), you need: a privacy policy page, explicit consent checkbox, and an unsubscribe mechanism. Use a service like Mailchimp or ConvertKit — don't store emails in your own database without proper security.
✓ Privacy policy linked in site footer, consent checkbox on forms
REQUIRED
S6
Content Moderation Guardrails
GUARDIAN (§09) is your primary content filter. Additionally, instruct all agents to refuse: hateful content, misinformation, PII (personal info), competitor defamation, and TOS-violating platform content. ARIA flags TOS issues before GUARDIAN reviews.
✓ Test: send MATRIX a harmful topic. It should decline and explain why.
REQUIRED

§11
Failure Protocols & 3-Strike Rule

Failures happen. How you respond determines whether they become learning moments or pipeline shutdowns. Every failure is documented, categorized, and escalated.

The 3-Strike Rule

📋 Definition: A "strike" is when any single agent fails to produce acceptable output (rejected by GUARDIAN or fails its own quality check) on the same task type, 3 consecutive times. Three strikes = mandatory escalation. The task is flagged, documented, and the pipeline pauses on that task type until the root cause is resolved.

Failure Categories

🔴
CRITICAL — Pipeline Down
API endpoint returning 500 errors. GUARDIAN permanently blocking all output. Site offline. Immediate action required — do not stream until resolved.
🟡
WARNING — Degraded Output
GUARDIAN scores consistently between 40–60. SPARK scores below 28 on multiple runs. Content is technically approved but underperforming. Investigate system prompt quality.
🔵
MINOR — Isolated Issue
Single pipeline run failed. One piece of content blocked. One agent produced off-brand output. Log it, regenerate, move on. No escalation needed unless it becomes a pattern.

If you receive a 429 (rate limit) error from Anthropic or OpenAI: wait 60 seconds → retry once → if still failing, switch to fallback model → log the incident → check billing dashboard for unusual usage spikes.

§12
Glossary

Plain-English definitions for every technical term used in this playbook. Bookmark this section for quick reference.

Agent
An AI model given a specific role, system prompt, and set of rules. Agents don't act on their own — they respond to inputs and produce outputs within their defined scope.
API
Application Programming Interface. A way for your website to talk to an external service (like Claude or GPT-4o). You send a request, they send back a response.
API Key
A secret password that proves to an API that you're authorized to use it. Treat like a credit card number — never share or commit to public code.
Backoff (Exponential)
A retry strategy where each failed attempt waits longer than the last: 2s, then 4s, then 8s. Prevents hammering an overloaded server.
CI/CD
Continuous Integration / Continuous Deployment. When you push code to GitHub, Netlify automatically builds and deploys it. Push to main = live in <60 seconds.
CORS
Cross-Origin Resource Sharing. A browser security rule that controls which websites can make API requests to your server. You configure it in netlify.toml.
Environment Variable
A secret value stored outside your code (in Netlify's dashboard). Your code reads it at runtime without it ever appearing in your codebase.
JAMstack
JavaScript + APIs + Markup. A modern web architecture where the frontend is static HTML/CSS/JS, and dynamic features are handled by APIs. Fast, secure, cheap to host.
Netlify Function
A serverless Node.js function that runs on-demand. Your AI API calls happen inside these — they're secure, scalable, and don't require a server to manage.
Pipeline
The sequential flow of work between agents: MATRIX → NOVA → SPARK → PULSE → ARIA → GUARDIAN. Output from each agent becomes input for the next.
Rate Limit
A cap on how many API requests you can make in a given time window. Anthropic and OpenAI both impose rate limits. You can also set your own on your API endpoint.
Serverless
Code that runs in the cloud without you managing a server. Netlify Functions are serverless — they spin up when needed and cost $0 when idle.
SPARK Score
SPARK's 40-point viral potential rating for content: Hook(10) + Retention(10) + Shareability(10) + CTA(10). Minimum passing score: 28/40.
System Prompt
The "personality and rules" you give to an AI agent before it processes your request. The most important part of building a reliable agent pipeline.
3-Strike Rule
If the same agent fails on the same task type 3 consecutive times, the pipeline pauses and that failure is formally documented and escalated for root-cause analysis.
TOS
Terms of Service. Every platform (Twitch, YouTube, TikTok, etc.) has rules about what content is allowed. ARIA flags TOS risks; GUARDIAN enforces compliance.
Variable Token
A placeholder like [BRAND_NAME] that you replace with your own value to customize this playbook for your brand. There are 10 template variables in total.
Viral Score
See SPARK Score. A quantified measure of how likely content is to spread organically based on hook strength, retention design, shareability, and CTA quality.
